변수

변수는 프로그램에서 센서로부터 읽은 값 혹은 계산에서 사용된 중간 값과 같은 값들을 나중에 사용하기 위하여 이름을 붙이고 저장하는 방법입니다.

변수 선언 하기

변수를 사용하기 전에, 변수를 선언하여야 합니다. 변수 선언이란 변수의 유형을 정의하고 선택적으로 초기 값을 설정하는 것(변수 초기화)을 의미합니다. 변수를 선언할 때 초기화(값을 지정)할 필요는 없지만 종종 유용합니다. 

int inputVariable1;
int inputVariable2 = 0;     // 둘 모두 맞습니다

프로그래머는 저장하여 사용하고자 하는 숫자의 크기를 고려하여 변수 유형을 선택하여야 합니다. 변수 값이 저장 용량을 초과한 값으로 저장되면 변수 값 뒤집어짐(rollover) 현상이 발생합니다. 밑에 나오는 예를 참고하세요.

변수 유효 범위

프로그래머가 직면한 또 다른 중요한 선택으로 변수를 선언할 위치를 정하는 것입니다. 변수들이 선언된 특정 위치에 따라 프로그램의 여러 함수들이 볼 수 있는(사용 하는) 변수에 대한 영향을 미칩니다. 이것을 변수 유효 범위라고 합니다.

변수 초기화 하기

변수는 변수를 선언할 때 초기화(시작 값을 넣는것)하거나 하지 않고 사용할 수 있습니다. 그렇지만 변수가 다른 용도로 사용되기 전에 유효한 데이터를 지니고 있는지 다시 확인하는 것이 항상 좋은 프로그래밍 습관입니다.

예: 

 int calibrationVal = 17;  // calibrationVal 변수를 선언하고 초기 값을 설정합니다

변수 값 뒤집어짐

변수를 변수가 가질 수 있는 최대 값을 넘게 만들면 변수의 최소 값으로 돌아가는 뒤집어짐(rollover) 현상이 발생합니다, 이러한 현상은 양쪽 방향으로 일어난다는 점을 알아두세요. 

   int x
   x = -32,768;
   x = x - 1;       // x는 지금 32,767 을 지니고 있습니다 - 음의 수에서 양의 수 방향으로 값 뒤집어짐 현상 발생


   x = 32,767;
   x = x + 1;       // x는 지금 -32,768 을 지니고 있습니다 - 값 뒤집어짐 현상 발생

변수 사용하기

변수가 선언되면, 대입 연산자(할당 연산자, '=' 단일 등 부호)를 사용하여 저장하려는 값과 변수를 같게 설정하여 사용합니다. 대입 연산자는 왼쪽에 있는 변수에 등 부호 오른쪽에 있는 무엇이든 넣도록 프로그램에 지시합니다. 

inputVariable1 = 7;             // inputVariable1 이름지어진 변수에 7을 넣습니다
inputVariable2 = analogRead(2); // inputVariable2 이름지어진 변수에 아날로그 2 번 핀에서
                                // 읽어들인 입력 전압(디지털 값으로 변환된)을 넣습니다

예

 int lightSensVal;
   char currentLetter;
   unsigned long speedOfLight = 186000UL;
   char errorMessage = {"다른 옵션을 선택하세요"}; // 문자열(string)을 참조하세요

변수가 설정(값이 지정)되면 값을 조사하여 특정 조건을 만족하는지 확인하거나 바로 값을 사용할 수 있습니다. 예로, 다음 코드는 inputVariable2 가 100 보다 작은지 여부를 시험한 다음 최소 값이 100인 inputVariable2으로 지연을 시킵니다: 

if (inputVariable2 < 100)
{
  inputVariable2 = 100;
}

delay(inputVariable2);

이 예에서 세가지 유용한 변수 사용법을 볼 수 있습니다. 이는 변수 값을 시험하고 ( if (inputVariable2 < 100) ), 시험을 통과하면 값을 설정하고 ( inputVariable2 = 100 ), delay() 함수의 입력 매개 변수로 변수의 값을 사용하는 것들입니다 ( delay(inputVariable2) )

스타일 노트: 코드를 보다 쉽게 읽을 수 있도록 변수에 변수를 설명하는 이름을 사용하여야 합니다. tiltSensor 혹은 pushButton 과 같은 변수 이름들은 코드를 읽는데 있어 변수가 무엇을 나타내는지 이해를 돕습니다. 반대로 var 혹은 value 와 같은 변수 이름들은 코드를 읽을 때 도움을 주지 않습니다.

아두이노에서 사용하고 있는 키워드가 아닌 단어로 변수 이름을 사용할 수 있습니다. 숫자로 시작하는 변수 이름을 사용하지 마세요.
