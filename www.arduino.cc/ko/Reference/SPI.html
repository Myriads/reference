SPI 라이브러리

이 라이브러리를 이용하여 아두이노를 마스터 장치로 동작시켜 다른 SPI 장치들과 통신할 수 있습니다.

직렬 주변기기 인터페이스에 대한 간단한 소개

직렬 주변기기 인터페이스(SPI: Serial Peripheral Interface)는 마이크로 컨트롤러가 짧은 거리에서 하나 이상의 주변 장치들과 신속하게 통신하기 위하여 사용하는 동기식 직렬 데이터 프로토콜입니다. 이것은 두 개의 마이크로 컨트롤러 간의 통신에도 사용할 수 있습니다.


SPI 연결에선 항시 마스터 장치(일반적으로 주변 기기들을 제어하는 마이크로컨트롤러)가 있습니다. 일반적으로 모든 장치에 3개의 공통 연결선들이 있습니다:

  • MISO (Master In Slave Out) - 슬레이브의 출력 마스터로 데이터를 보내는 용도,
  • MOSI (Master Out Slave In) - 마스터의 출력선 슬레이브로 데이터를 보내는 용도,
  • SCK (Serial Clock) - 마스터가 발생하는 클럭 펄스로 데이터 전송을 동기화.

그리고 모든 장치마다 다르게 할당된 연결선:

  • SS (Slave Select) - 마스터가 슬레이브 특정 장치를 선택하거나 선택 하지 않는 용도로 사용할 수 있는 각 장치의 핀입니다.

슬레이브 선택 핀이 LOW이면 마스터와 통신을 하고 HIGH 마스터 신호들을 무시합니다. 이렇게 하여 동일한 MISO, MOSI 그리고 CLK 선들을 공유한 여러 SPI 장치들을 사용할 수 있습니다.


새로운 SPI 장치용 코드를 작성하려면 몇 가지 사항을 알아야 합니다:

  • 장치에서 사용할 수 있는 최대 SPI 속도는 얼마인지? 이것은 SPISettings의 첫 번째 매개 변수에 의하여 제어됩니다. 만약 15MHz 급의 칩을 사용하는 경우 15000000을 사용합니다. 아두이노는 SPISettings에서 사용한 숫자와 같거나 적은 최상의 속도를 자동으로 사용합니다.
  • 데이터가 MSB(Most Significant Bit) 또는 LSB(Least Significant Bit)가 먼저 내 보내집니까? 이것은 두 번째 SPISettings 매개 변수인, MSBFIRST 혹은 LSBFIRST로 제어됩니다. 대부분의 SPI 칩들은 MSB가 최초인 데이터 순서를 사용합니다.
  • 데이터 클럭이 HIGH 혹은 LOW일 때 쉬는 시간인지? 클럭 파형의 상승 또는 하강 에지에서 샘플을 하는지? 이러한 모드들은 SPISettings의 세 번째 매개 변수에 의하여 제어됩니다.

SPI 표준은 엄격하지 않아(느슨하여) 각 장치는 조금씩 다르게 구현됩니다. 이것은 코드를 작성할 때 장치의 설명서에 각별한 주의가 필요하다는 것을 의미합니다.

일반적으로 네가지 전송 모드가 있습니다. 데이터 클럭 신호의 상승 또는 하강 에지에서 데이터를 내 보내거나 받아들일지(클럭 위상(phase)이라고 함), 그리고 클럭이 HIGH 혹은 LOW에서 유휴 상태인지(클럭 극성(polarity)이라고 함) 여부를 제어합니다. 다음 표에 따라 네 가지 모드는 극성과 위상을 겹합합니다:

SPI 매개 변수들이 갖춰졌다면 SPI.beginTransaction()으로 SPI 포트 사용을 시작합니다. SPI 포트는 모든 설정 정보 값들로 구성될 것입니다. 가장 간단하고 효율적인 방법은 SPISettings을 바로 SPI.beginTransaction() 안에 사용하는 것입니다. 예로:

SPI.beginTransaction(SPISettings(14000000, MSBFIRST, SPI_MODE0));

다른 라이브러리가 인터럽트에서 SPI를 사용하면, SPI.endTransaction()을 호출할 때까지 SPI를 액세스할 수 없습니다. SPI 설정은 트랜잭션 시작시 적용되며 SPI.endTransaction()은 SPI 설정을 변경하지 않습니다. 일부 라이브러리가 beginTransaction()을 두 번 호출 않는 다면, 설정은 유지됩니다. 프로그램을 SPI를 사용하는 다른 라이브러리와 함께 사용할 경우 최고의 호환성을 위하여 SPI.endTransaction() 호출하기 전의 시간을 최소로 하여야 합니다.

대부분의 SPI 장치를 사용할 때, SPI.beginTransaction() 후에, 슬레이브 선택 핀을 LOW로 하고, 데이터를 전송하기 위하여 SPI.transfer()을 원하는 횟수만큼 호출하고, SS 핀을 HIGH로 만들고 마지막으로 SPI.endTransaction()을 호출합니다.

SPI에 대한 자세한 사항은 위키 백과의 SPI 페이지를 참조하세요.

MISO, MOSI와 SCK 신호들은 ICSP 헤더의 항시 같은 핀 위치에서 사용할 수 있습니다. 예를 들어 모든 보드에서 동작하는 실드를 설계할 때 유용합니다.

AVR 기반 보드들에서 슬레이브 선택 SS 핀에 대한 노트

모든 AVR 기반 보드들은 외부 마스터에 의하여 제어되는 슬레이브로 동작할 때 사용 하기 편리한 SS 핀을 가지고 있습니다. 이 라이브러리는 마스터 모드만 지원하기 때문에 이 핀을 항시 OUTPUT으로 설정하여야 합니다. 그렇지 않으면 SPI 인터페이스는 하드웨어에 의하여 자동으로 슬레이브 모드로 들어 가게 되어 라이브러리가 동작하지 않게 됩니다.

그렇지만 장치의 슬레이브 선택 SS 핀을 어떠한 디지털 핀들을 사용할 수 있습니다. 예로, 아두이노 이더넷 실드는 4번 핀을 보드에 내장한 SD 카드 SPI 연결 제어에 사용하고 10번 핀을 이더넷 제어기에 사용하고 있습니다.
