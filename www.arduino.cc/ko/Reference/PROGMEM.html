PROGMEM

SRAM 메모리 대신에 플래시(Flash, 프로그램) 메모리에 데이터를 저장하는데 사용합니다. 아두이노 보드에서 사용 할 수 있는 여러 메모리 유형들에 대한 설명이 있습니다.

PROGMEM 예약어(열쇠글, 키워드, keyword)는 변수 수정자로 pgmspace.h에 정의된 데이터 유형들과 같이 사용하여야 합니다. PROGMEM 예약어는 컴파일러에게 SRAM 대신 "이 정보를 플래시 메모리에 저장한다"고 알려 줍니다.

PROGMEM은 AVR 아키텍처에서만 사용할 수 있는 pgmspace.h 라이브러리의 부분입니다. 그렇기 때문에 스케치 맨 앞에 다음과 같이 라이브러리를 포함하여야 합니다:

#include <avr/pgmspace.h>
[코드 얻기]

문법

const dataType variableName[] PROGMEM = {data0, data1, data3...};
[코드 얻기]

  • dataType - 모든 변수 유형
  • variableName - 데이터 배열 이름

PROGMEM은 변수 수정자이기 때문에 어떤 곳에 놓아야만 하는 엄격하고 빠른 규칙이 없어 아두이노 컴파일러는 밑의 모든 정의를 벋아들여, 같이 사용할 수 있습니다. 그렇지만 다양한 버전의 아두이노(GCC 버전과 관련이 있음)에서 실험에서, PROGMEM이 한 위치에서는 동작하고 다른 위치에서는 동작하지 않을 수 있음을 보여주고 있습니다. 아래의 "문자열 테이블" 예는 아두이노 IDE 버전 13에서 동작을 시험하였으며 이전 버전의 IDE는 변수 이름 뒤에 PROGMEM이 변수 이름 뒤에 놓여 있으면 보다 잘 동작합니다.

const dataType variableName[] PROGMEM = {};   // 이 양식을 사용하세요
const PROGMEM  dataType  variableName[] = {}; // 혹은 이 양식도 사용하세요
const dataType PROGMEM variableName[] = {};   // 이 양식은 사용하지 마세요
[코드 얻기]

PROGMEM을 한 변수에 사용할 수 도 있지만, 저장할 아주 큰 데이터 블록이 있는 경우 매우 유용합니다. 일반적으로 가장 쉽게 사용할 수 있는 배열과 현재 논의되고 있지는 않지만 C 데이터 구조체 struct가 있습니다.

PROGMEM을 사용하는 것도 2 단계 절차입니다. 데이터를 플래시 메모리에 저장한 후, 프로그램 메모리로부터 SRAM으로 다시 읽기 위하여 pgmspace.h 라이브러리에 정의된 특별한 방법(함수)이 필요하며. 읽어와 유용하게 사용할 수 있습니다.

예

다음 코든 조각은 char(1 바이트) 데이터들과 int(2 바이트) 데이터들을 PROGMEM에 넣고, 읽어 들이는 방법을 보여 줍니다.

#include <avr/pgmspace.h>


// 부호 없는 정수들을 저장합니다
const PROGMEM  uint16_t charSet[]  = { 65000, 32796, 16843, 10, 11234};

// 일부 문자들을 저장합니다
const char signMessage[] PROGMEM  = {"I AM PREDATOR,  UNSEEN COMBATANT. CREATED BY THE UNITED STATES DEPART"};

unsigned int displayInt;
int k;    // 카운터 변수
char myChar;


void setup() {
  Serial.begin(9600);
  while (!Serial);

  // 초기화 코드를 여기에 넣으세요, 한번만 실행됩니다:
  // 2 바이트 int 정수를 다시 읽어 들입니다
  for (k = 0; k < 5; k++)
  {
    displayInt = pgm_read_word_near(charSet + k);
    Serial.println(displayInt);
  }
  Serial.println();

  // 문자를 읽어 들입니다
  int len = strlen_P(signMessage);
  for (k = 0; k < len; k++)
  {
    myChar =  pgm_read_byte_near(signMessage + k);
    Serial.print(myChar);
  }

  Serial.println();
}

void loop() {
  // 여기에 주 처리 코드를 넣으세요, 반복 실행됩니다:

}

문자열 배열

LCD 디스플레이를 이용하는 프로젝트와 같이 많은 양의 텍스트로 작업할 때, 문자열의 배열을 만들어 사용하는 것이 편리합니다. 문자열 자체 또한 배열이므로, 실제로는 2 차원 배열의 예입니다.

이것들은 큰 구조라 프로그램 메모리에 저장하는 것이 바람직합니다. 다음 코드는 아이디어를 설명합니다.

/*
 PROGMEM 문자열 데모
 문자열 테이블을 프로그램 메모리 (플래시)에 저장하고,
 저장한 것들을 꺼집어 내는 방법.

 다음으로부터 요약된 정보:
 http://www.nongnu.org/avr-libc/user-manual/pgmspace.html

 프로그램 메모리에 문자열 테이블(배열)을 만드는 것은 약간 복잡하지만,
 여기에 따라 해 볼 좋은 본보기가 있습니다.

 문자열 테이블은 2 단계로 이뤄집니다. 먼저 문자열들을 정의합니다.
*/

#include <avr/pgmspace.h>
const char string_0[] PROGMEM = "문자열 0";   // "문자열 0" 등은 저장할 문자열입니다 - 맞게 수정하세요.
const char string_1[] PROGMEM = "문자열 1";
const char string_2[] PROGMEM = "문자열 2";
const char string_3[] PROGMEM = "문자열 3";
const char string_4[] PROGMEM = "문자열 4";
const char string_5[] PROGMEM = "문자열 5";


// 그리고 문자열을 참조하는 테이블을 만듭니다.

const char* const string_table[] PROGMEM = {string_0, string_1, string_2, string_3, string_4, string_5};

char buffer[30];    // 가장 긴 문자열을 넣을 수 있는 충분한 크기인지 확인합니다

void setup()
{
  Serial.begin(9600);
  while(!Serial);
  Serial.println("OK");
}


void loop()
{
  /* 프로그램 메모리에서 문자열 테이블을 사용하려면 특수 함수를 사용하여
     데이터를 읽어 들여야 합니다.
     strcpy_P 함수는 프로그램 공간의 문자열을 RAM("buffer")으로 복사합니다.
     RAM으로 받아 들이는 문자열이 프로그램 공간에서 읽어들인 내용을 저장할 수
     있을 만큼 큰지 확인하세요. */


  for (int i = 0; i < 6; i++)
  {
    strcpy_P(buffer, (char*)pgm_read_word(&(string_table[i]))); // 유형 변환(cast)과 참조 필요, 그리고 복사.
    Serial.println(buffer);
    delay( 500 );
  }
}

노트

PROGMEM을 사용하려면 변수를 전역 변수로 정의하거나 static 예약어로 정의하여야 합니다.

다음 코드는 함수 안에서 작동 하지 않습니다:

const char long_str[] PROGMEM = "Hi, I would like to tell you a bit about myself.\n";
[코드 얻기]

다음 코드는 함수 안에 지역 변수로 정의 되었지만 작동 할 것입니다:

const static char long_str[] PROGMEM = "Hi, I would like to tell you a bit about myself.\n"

F() 매크로

다음과 같은 명령어:

Serial.print("시리얼 모니터로 출력할 내용");
[코드 얻기]

를 사용하면, 출력할 문자열은 일반적으로 RAM에 저장됩니다. 시리얼 모니터로 많은 내용을 출력하면, RAM을 쉽게 모두 사용하여 메모리가 부족할 수 있습니다. 플래시 메모리에 빈 공간이 있다면, 다음 구문을 사용하여 문자열을 플래시에 저장하여야 한다는 것을 쉽게 나타낼 수 있습니다:

Serial.print(F("플래시에 저장된 시리얼 모니터로 출력할 내용"));
[코드 얻기]
