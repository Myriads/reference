두에 확장 SPI 사용법

새로운 프로젝트에서는 이 함수를 사용하지 마세요. SPI.beginTransaction()에 SPISettings를 사용하여 SPI 파라메터들을 설정할 수 있습니다.

아두이노 두에(Due)에 사용된 SAM3X는 향상된 SPI 기능들을 지니고 있습니다. 확장된 사용법과 AVR 기반의 사용법을 모두 사용할 수 있습니다.

확장된 API는 핀들 4, 10, 그리고 52를 CS(슬레이브 칩 선택)로 사용할 수 있습니다.

사용법

SPI 장치에 사용할 CS로 사용할 각 핀을 지정하여야 합니다.

두에는 SPI 버스를 공유하는 여러 장치들 간에 칩 선택을 자동으로 다룰(처리 할) 수 있습니다. 각 장치들은 또한 각기 다른 속도와 데이터 모드와 같은 속성들을 지닐 수 있습니다.

다른 CS 핀들을 이용하는 여러 장치들을 사용할 경우, setup()에서 사용할 핀들을 선언 하여야 합니다. 다음 예에서 SPI MISO, MOSI 그리고 SCK 핀들을 공유하는 두 개의 장치가 있습니다. 한 장치는 CS를 핀 4에 연결 하고, 다른 장치는 핀 10에 연결 하였습니다.

void setup(){
  // 핀 4에 CS가 연결된 장치의 버스를 초기화 합니다
  SPI.begin(4);
  // 핀 10에 CS가 연결된 장치의 버스를 초기화 합니다
  SPI.begin(10);
}
[코드 얻기]

일단 핀이 CS 핀으로 선언되면, 기본 동작 또한 변경할 수 있습니다. 예를 들어 장치들이 각기 다른 클럭 속도로 동작하는 경우, setup()은 이와 같을 수 있습니다:

void setup(){
  // 핀 4에 CS가 연결된 장치의 버스를 초기화 합니다
  SPI.begin(4);
  // 핀 4에 연결된 장치의 클럭 분배기 값을 21로 설정합니다
  SPI.setClockDivider(4, 21);
  // 핀 10에 CS가 연결된 장치의 버스를 초기화 합니다
  SPI.begin(10);
  // 핀 10에 연결된 장치의 클럭 분배기 값을 84로 설정합니다
  SPI.setClockDivider(10, 84);
}
[코드 얻기]

핀 4에 연결된 장치로 단일 바이트 전송은 이와 같을 수 있습니다:

void loop(){
  byte response = SPI.transfer(4, 0xFF);
}
[코드 얻기]
위에서, “0xFF”는 핀 4에 연결된 SPI 장치로 보내지고 MISO로 들어오는 데이터는 변수 response에 저장됩니다. 칩 선택은 SPI 제어기(컨트롤러)에 의하여 자동으로 처리되며, transfer 명령은 다음을 의미합니다:

  • 핀 4를 LOW로 하여 칩(슬레이브 장치)을 선택합니다
  • SPI 버스로 0xFF를 내 보내며 받은 데이터를 반환합니다
  • 핀 4를 HIGH로 하여 장치 선택을 해제합니다

transfer 명령에 전송 후 SPI 장치를 해제하지 않도록 알려 한 트랜잭션에서 두 바이트 이상의 데이터를 보낼 수 있습니다:

void loop(){
// 핀 10의 장치에 0x0F를 보내고, 칩 선택을 유지합니다
SPI.transfer(10, 0xF0, SPI_CONTINUE);
// 핀 10의 장치에 0x00를 보내고, 칩 선택을 유지합니다
SPI.transfer(10, 0×00, SPI_CONTINUE);
// 핀 10의 장치에 0x00를 보내고, response1에 받은 결과를 저장하고, 칩 선택을 유지합니다
byte response1 = SPI.transfer(10, 0×00, SPI_CONTINUE);
// 핀 10의 장치에 0x00를 보내고, response2에 받은 결과를 저장하고, 칩 선택을 해제합니다
byte response2 = SPI.transfer(10, 0×00);
}
[코드 얻기]
매개 변수 SPI_CONTINUE는 전송 간에 칩 선택이 활성화된 상태로 유지되도록 합니다. 마지막 전송에 SPI_CONTINUE가 지정하지 않음으로 마지막 데이터로 전송되게 합니다.

확장된 방법을 사용할 때 올바른 문법은 setClockDivider(), setDataMode(), transfer(), setBitOrder() 각각의 참조 페이지들을 참고하세요.

주의 : SPI.begin()이 호출되면, 선언된 pin은 일반 용도의 I/O 핀으로 사용할 수 없습니다
