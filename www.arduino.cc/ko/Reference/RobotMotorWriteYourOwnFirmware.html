Robot

자신의 펌웨어 만들기

로봇 모터 보드는 "상태 머신(state machine)"으로 동작하도록 만들어져 있어, 로봇 제어 보드로부터 명령들을 받아 그에 따라 실행합니다. 또한 요청시 데이터를 돌려 줍니다(반환 합니다). 기본 모터 보드의 펌웨어는 그렇게 많은 지능을 지니고 있지 않아 대부분의 응용 프로그램에 맞게 조정할 필요는 없습니다.

그렇지만 자신의 모드를 정의하거나 혹은 자신만의 명령을 만들어 모터 보드의 기능을 확장할 수 있습니다. 이렇게 하기 위해선 로봇 제어 보드 라이브러리 파일들을 수정하여야 합니다.

이 지침들은 이미 프로그래밍에 익숙한 분들을 위한 것입니다.

모드들

로봇은 모터 보드가 어떻게 작동하여야 하는지를 알려주는 미리 정의된 몇가지 모드들이 있습니다. 예를들어, "선 추적 모드"가 있으며, 이 모드로 들어가면, 모터 보드 밑에 탑재된 적회선 센서 배열로부터 값을 읽어 로봇이 가고자 하는 방향을 계산하고 모터를 바로 조정합니다.

제어 보드에서 setMode()를 사용하여 모드를 변경할 수 있습니다.

자신만의 모드를 정의할 수 있으며, 새로운 모드를 만들기 위하여, 제어 보드와 모터 보드 두 보드에 모드를 정의하여야 합니다. 또한 원하는 동작을 위하여 로직을 만들어야 합니다.

새로운 모드 정의 하기

  • OS에서, 아두이노 IDE가 설치되어 있는 폴더로 이동합니다.
  • 사용하는 텍스트 편집기로 "libraries/Robot_Control/src/ArduinoRobot.h" 와 "libraries/Robot_Motor/src/ArduinoRobotMotorBoard.h"을 엽니다.
  • 각각의 파일에서, 모터 모드가 정의되어 있는 부분을 찾습니다. 다음과 같이 보일 것입니다:

//motor board modes
#define MODE_SIMPLE 0
#define MODE_LINE_FOLLOW 1
#define MODE_ADJUST_MOTOR 2
#define MODE_IR_CONTROL 3
[코드 얻기]

  • 두 파일 모두 이것들 아래에 새로운 모드를 다음과 같이 추가합니다: "#define MODE_MY_MODE_NAME num". num은 다른 모드들과 다른 숫자이어야 합니다.
  • ArduinoRobotMotorBoard.h 과 ArduinoRobot.h을 저장하고 닫습니다
  • "libraries/Robot_Motor/src/ArduinoRobotMotorBoard.cpp"를 엽니다
  • "process()" 정의를 찾고 모드 이름에 해당하는 새 조건을 추가합니다. 여기에 있는 코드들이 loop()에서 수행되게 되며, 스케치에서와 같이 여기에 로직을 작성하면 됩니다.
  • ArduinoRobotMotorBoard.cpp을 저장하고 닫습니다.
  • 이제 로봇 제어 보드의 스케치에서, "setMode(MODE_MY_MODE_NAME )"를 사용하여 새로운 모드로 바꿀 수 있습니다

명령들

두가지 유형의 명령들이 있습니다:

  • 지시 (1-방향 통신) : motorsWrite()와 비슷하게, 제어 보드는 명령을 보내고, 명령을 모터 보드가 받아 이를 수행합니다. 명령을 보낸 보드로 아무 것도 보내지지 않습니다.

  • 질의 (2-방향 통신) : updateIR()과 비슷하게, 제어 보드는 모터 보드로 메시지를 보내고, 제어 보드로 데이터를 돌려 줍니다.

새로운 명령을 만들기 위하여, 명령 코드를 정의하여야 하고 이에 대한 보내기/받기 코드를 작성하여야 합니다.

새로운 명령 정의 하기

  • OS에서, 아두이노 IDE가 설치되어 있는 폴더로 이동합니다.
  • "libraries/Robot_Control/src/ArduinoRobot.h" 와 "libraries/Robot_Motor/src/ArduinoRobotMotorBoard.h" 두 파일을 엽니다
  • 선택한 텍스트 편집기로 "libraries/Robot_Control/src/ArduinoRobot.h"와 "libraries/Robot_Motor/src/ArduinoRobotMotorBoard.h"를 엽니다.
  • 각각의 파일에서, 다음과 같이 보이는 명령 정의 부분을 찾습니다:

//Command code
#define COMMAND_SWITCH_MODE 0
#define COMMAND_RUN 10
#define COMMAND_MOTORS_STOP 11
..........
#define COMMAND_READ_TRIM_RE 81
#define COMMAND_PAUSE_MODE 90
#define COMMAND_LINE_FOLLOW_CONFIG 100
[코드 얻기]

  • 이렇게 두 파일 모두 새 명령을 뒤에 추가합니다: "#define COMMAND_MY_COMMAND_NAME num". num은 다른 명령들과 다른 숫자여야 합니다.
  • 만약 명령이 2-방향 질의이면, 여기에 다른 명령 코드를 정의합니다:"#define COMMAND_MY_COMMAND_NAME_RE num1" 이것은 반환 명령에 사용됩니다. "num1"은 위의 "num"과 다른 숫자입니다. /li>
  • 명령의 구조를 계획합니다. 명령을 저장하기 위하여 byte 혹은 int를 사용할 수 있습니다. 명령을 보내고/받을 때, 해당 데이터를 쓰고/읽는 것은 같은 순서로 이뤄져야 합니다. 예로 "A message structure will be:"로 시작하는 주석들을 보기 바랍니다.
  • 명령을 보내기 위하여, RobotControl 라이브러리에 함수를 만들어야 하며, 메시지를 보내는 방법으로, motors.cpp에 있는 "motorsWrite()" 함수나 sensors.cpp에 있는 "updateIR()"를 참조하세요.
  • 명령이 어떤 명령인지 알아내려면, ArduinoRobotMotorBoard.cpp에 있는 "parseCommand()"를 찾아, 앞에서 정의한 명령과 동일한 이름의 case 절에 새 조건을 추가하세요. 다른 조건들을 예로 삼아 참조하세요.
  • 명령이 지시라면, 논리를 수행하는 함수를 작성하면 모든 것이 완료됩니다. 만약 질의라면 명령을 실행하는 함수와 요청한 정보를 반환하는 함수를 만들어야 합니다. 예로 ArduinoRobotMotorBoard.cpp"에 있는 _digitalRead()"를 참조하세요.
